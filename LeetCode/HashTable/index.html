Hash Table

哈希表是根据关键码的值而直接进行访问的数据结构。

eg：array、set、map

哈希函数

通过hashCode把元素转化为数值，然后映射到哈希表中

哈希碰撞

两个不同的元素映射到了同一个位置（因为哈希表也是有长度的，所以有可能出现）

解决哈希碰撞的两种方法：拉链法|线性探测法

拉链法

在哈希表的每个元素上指向一个链表，用链表存储发生碰撞的元素

线性探测法

简而言之，当这个位置已经有元素了，那我放下一个位置不就好了，下一个同理。所以就要求数据量要小于哈希表的大小

应用场景

当我们遇到要快速判断一个元素是否出现在集合中|一个元素是否出现过时，就可以考虑哈希法。
